## Phase 1: 基本フロー (作成順)

### 1-1. 停留所マスタの作成

チャットに打つ:

```
20241107築上町コミバス停留所情報.csv から停留所を作って
```

確認:

- [ ] 確認画面が出る
- [ ] タイトル「**1. 停留所情報の取り込み（CSV）**」
- [ ] サマリ:「**20241107...から 停留所マスタ (stops.txt) を作成します。**」
- [ ] 「✓ 実行する」を押す
- [ ] AI 応答に「stops.txt を生成しました (75 件の停留所)」
- [ ] サイドバー「📁 作成中のファイル」に `stops.txt` が現れる
- [ ] 「🗺 生成結果のプレビュー」が現れ、「📍 停留所」を展開すると **地図に 75 マーカー** が出る

### 1-2. 時刻表の作成 (整形済 CSV)

チャットに打つ:

```
築城巡回線右回り.csv から時刻表を作って
```

確認:

- [ ] 確認画面が出る
- [ ] タイトル「**1. 時刻表データの作成**」
- [ ] サマリ:「**築城巡回線右回り.csv から 時刻表データ (stop_times.txt) を作成します。**」
- [ ] **「📊 変換後の時刻表 (確認用)」テーブル** が表示される
- [ ] cross-file 警告が **6 件** 出る:
  - [ ] 🔀 似た名前が登録されています: 椎田交番、高塚、ルミエール前、八津田
  - [ ] ❌ 停留所情報に存在しません: 築上町図書館、柳渡橋
- [ ] 「✏ stops.txt にその場で追加 (6 件)」 expander が出ている

### 1-3. 復旧フロー: 表記揺れ 4 件を解消

「✏ stops.txt にその場で追加」を展開:

#### 椎田交番 (類似名: 椎田交番（上り）、椎田交番（下り）)

- [ ] 緯度経度が**自動入力されている** (上り の座標がコピーされている)
- [ ] 「➕ 追加」を押す → 「✓ 椎田交番 を stops.txt に追加 (stop_id = S<N>)」
- [ ] 自動で画面が再描画され、警告件数が **6 → 5 件** に減る

#### 高塚

- [ ] 緯度経度が自動入力 (`高塚（上り）` の座標)
- [ ] 「➕ 追加」 → 警告 5 → 4 件

#### ルミエール前

- [ ] 緯度経度が自動入力
- [ ] 「➕ 追加」 → 警告 4 → 3 件

#### 八津田

- [ ] 緯度経度が自動入力
- [ ] 「➕ 追加」 → 警告 3 → 2 件

### 1-4. 復旧フロー: 真の登録漏れ 2 件に対処

#### 築上町図書館

- [ ] 緯度経度が**空白** (類似名が無いため)
- [ ] Google マップ等で「築上町図書館」を検索して座標を取得 (例: 緯度 33.640, 経度 131.041)
- [ ] 緯度経度を入力 → 「➕ 追加」 → 警告 2 → 1 件

#### 柳渡橋

- [ ] 同様に座標を入力 → 「➕ 追加」 → 警告 1 → 0 件

### 1-5. 確認画面で実行

- [ ] 警告が完全に消えた状態で「✓ 実行する」
- [ ] AI 応答に「stop_times.txt を生成しました (CSV, 137 行)」
- [ ] **未解決の警告メッセージは出ない** (= 全 stop_id が解決された)
- [ ] 「🗺 生成結果のプレビュー」→「⏰ 時刻表」を展開すると 137 行の表が見える、全行に `stop_id` が入っている

### 1-6. KML から座標抽出

チャットに打つ:

```
築城巡回線（右回り）築上町コミュニティバス.kml から座標を抽出して
```

- [ ] サイドバー「📁 作成中のファイル」に `築城巡回線（右回り）..._から_..._へのルート.txt` が 8 個追加される

### 1-7. shapes.txt 生成

チャットに打つ:

### 1-8. ZIP ダウンロード

- [ ] サイドバー「📥 GTFS データを ZIP でダウンロード」ボタンが緑 (有効)

## Phase 2: ハルシネーション抑制 (本研究の主目的)

「会話をリセット」を一度押してから始める。

### 2-1. ファイル名なしで時刻表ツール呼び出し

チャットに打つ:

```
stop_times を作って
```

期待:

- [ ] AI が「どの時刻表ファイルから作りますか?」のように **質問** する
- [ ] 勝手に「サンプルから」「適当なファイルから」と決めない
- [ ] 確認画面に進まない (= 引数欠落の段階で対話継続)

### 2-2. shapes ツールで引数不足

チャットに打つ:

```
shapes.txt を作って
```

期待:

- [ ] AI が **shape_id と座標ファイル名の両方** を質問する
- [ ] 片方だけ聞いて確認画面に進む、ということはない

### 2-3. 存在しないツール

チャットに打つ:

```
停留所の座標を入れて stops.txt を作って
```

期待:

- [ ] AI が「そのツールは無い」「座標は元データに記載のものを使う」のような応答
- [ ] **緯度経度を AI が捏造して確認画面に出す、ということは無い**

- 複数セグメント KML の自動結合は future work
